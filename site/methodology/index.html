Methodology

Reference construction protocol and calculation standards.

  1. Intraday collection writes timestamped attempts to /intraday/YYYY-MM-DD/HH-MM-SS.json.
  2. Publication window remains 13:45-14:15 UTC.
  3. Official daily publication selects the latest valid intraday attempt in that window.
  4. If the latest attempt is invalid, publication falls back to the most recent valid attempt.
  5. If no valid attempts exist, the day is published as WITHHOLD (no fabricated rate).
  6. For each selected attempt, FIX is computed from source medians; BAND is p25-p75; DISPERSION is (p75 - p25) / FIX.
  7. WITHHOLD can still occur even when a numeric candidate exists (for example stale inputs or high dispersion).
  8. Status bands: Green <= 1.5%, Amber <= 3.5%, Red <= 5%.
  9. WITHHOLD is triggered by invalid/stale inputs, insufficient source quality, or excessive dispersion.
  10. Derived indicators are suppressed unless required benchmark inputs are available; Street–Mobadeleh Gap and Crypto Premium may use the Street candidate value when Street is WITHHOLD.

Managed-Market Series Transition

  • NIMA legacy status: NIMA was a legacy platform and RialWatch does not currently maintain a historical NIMA series in the published dataset.
  • Successor managed-market series: from 2025-01-20 onward, the live dashboard uses Exchange Center USD Remittance Rate as the current managed-market benchmark track.
  • Unified remittance quote transition: after the early-January-2026 hall merger, the successor series is treated as a unified commercial/remittance managed-market quote when source structure reflects that merge.
  • Current benchmark mapping: Mobadeleh USD Rate (mapped to navasan: mob_usd) represents the live managed-market benchmark in production.
  • Integrity guardrails: no guessed NIMA ticker is used and no fabricated values are published.
  • Regional Transfer Rate remains production-approved with strict canonical source mapping.
