Methodology

How RialWatch defines, validates, and publishes its USD/IRR benchmark.

1. Benchmark Definition

Primary benchmark: Open Market / Street USD/IRR daily benchmark.

RialWatch publishes one official daily benchmark value, not a live ticker. Daily publication is either Published or Withheld based on validation rules.

Supplementary cards (Official Commercial, Regional Transfer, Crypto USDT, Emami Gold Coin) are separate readings and do not replace the primary benchmark.

2. Market Structure

  • Open Market / Street Rate is the flagship public benchmark.
  • Official Commercial USD Rate is the managed-market anchor currently mapped to navasan:mex_usd_sell.
  • Regional Transfer, Crypto Dollar (USDT), and Emami Gold Coin are tracked as distinct market channels/signals.
  • NIMA is treated as a legacy mechanism and is not currently maintained as a live standalone series.

3. Data Normalization

  • All stored benchmark values are normalized to IRR (rial) internally.
  • If a source provides a value in toman, RialWatch converts using rial = toman * 10.
  • If a source provides a value in rial, it is stored unchanged.
  • Invalid, missing, non-numeric, stale, or out-of-window values are excluded from benchmark computation.

4. Calculation Approach

  1. Intraday attempts are stored at timestamped paths: /intraday/YYYY-MM-DD/HH-MM-SS.json.
  2. Within each source, sampled values are reduced to a source median for the benchmark window.
  3. FIX is the median of source medians.
  4. BAND is the 25th-75th percentile range of source medians.
  5. DISPERSION is computed as (p75 - p25) / FIX.
  6. Indicator formulas (Street-Official, Street-Transfer, Street-Crypto, Official 7d Trend) are derived separately from benchmark values.

5. Publication Rules

  • Publication window: 13:45-14:15 UTC. Official publish checkpoint: 14:20 UTC.
  • Selection rule: latest valid intraday sample in-window; fallback to most recent valid in-window sample if needed.
  • Publish state is Withheld when methodology checks fail (for example insufficient valid sources, stale/invalid inputs, or excessive dispersion).
  • No fabricated values are published. Missing configuration is explicitly surfaced as CONFIG NEEDED.
  • Historical outputs are immutable once written; no silent backfill/rewrite of prior published days.
